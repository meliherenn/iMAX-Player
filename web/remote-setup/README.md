# Connected iMAX

Static companion site for optional TV playlist entry. The Android app keeps this feature disabled
unless both `REMOTE_SETUP_API_BASE_URL` and `REMOTE_SETUP_WEB_BASE_URL` are configured at build time.
No deployment, endpoint, provider account, or credential is bundled in this repository.

It supports:

- Xtream / Portal credentials
- M3U / M3U8 URL
- Local M3U file payload creation
- Optional EPG URL and startup/EPG preferences
- 8-character pairing-code based submission contract

Run locally:

```bash
cd web/remote-setup
python3 -m http.server 4173
```

Open:

```text
http://127.0.0.1:4173/?code=A7K9Q2BC
```

Provide the HTTPS API base in the query string used by the TV QR code:

```text
https://setup.example.com/?code=A7K9Q2BC&api=https%3A%2F%2Fsetup-api.example.com
```

The companion site posts:

```text
POST /api/pairings/{pairingCode}/playlist
```

Without `api`, it only builds the payload locally and allows copying it for integration testing.

## Required backend contract

The backend is intentionally not included. A production deployment must implement:

```text
POST /api/pairings
Body: { "pairingCode": "A7K9Q2BC", "status": "pending", "payload": {} }

GET /api/pairings/A7K9Q2BC
Response: { "status": "pending|completed", "payload": { ... } }

POST /api/pairings/A7K9Q2BC/playlist
Body: the Connected iMAX payload generated by app.js
```

Security requirements:

- HTTPS only outside localhost development.
- Pairings expire after at most 10 minutes and are deleted after retrieval.
- Rate-limit creation, polling, and submission endpoints.
- Never expose an endpoint that lists pairing rows or payloads.
- Bind submission and retrieval to the exact 8-character pairing code; use constant-time comparison where practical.
- Set strict CORS to the deployed companion-site origin.
- Do not log payloads because they can contain playlist URLs and Xtream credentials.
- Encrypt transient payloads at rest and publish a privacy notice naming the operator and retention period.
